Quicksort Algorithm

History

Quicksort was developed by Tony Hoare in 1960, originally as a sorting method that was much faster than the previously known sorting methods like Bubble Sort and Insertion Sort. Its primary advantage lies in its divide-and-conquer approach.

Quicksort Algorithm

Quicksort is a divide-and-conquer algorithm that works by selecting a 'pivot' element from the array and partitioning the other elements into two sub-arrays, according to whether they are less than or greater than the pivot.

Steps:

  • Pick an element as a pivot.
  • Partition the array into two subarrays – one with elements less than the pivot, the other with elements greater than the pivot.
  • Recursively apply the same process to the subarrays.

Code Example:

function quicksort(arr) {
    if (arr.length <= 1) return arr;
    let pivot = arr[arr.length - 1];
    let left = [], right = [];
    for (let i = 0; i < arr.length - 1; i++) {
        if (arr[i] < pivot) left.push(arr[i]);
        else right.push(arr[i]);
    }
    return [...quicksort(left), pivot, ...quicksort(right)];
}

Time Complexity Analysis

The average time complexity of Quicksort is O(n log n), making it one of the most efficient sorting algorithms. However, in the worst case, when the pivot always picks the smallest or largest element, the time complexity becomes O(n²).

Best Case: O(n log n)

Average Case: O(n log n)

Worst Case: O(n²)

Use Cases

Quicksort is widely used in practice due to its efficiency, especially for large datasets. It is commonly applied in databases, file systems, and other high-performance applications that require fast sorting.
